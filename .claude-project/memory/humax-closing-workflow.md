Humax 재무팀 고정비 월 결산 워크플로우 (`docs/rfp/클로드 코워크 활용안_고정비.xlsx` 기록).

## 7단계

1. **Raw Update** — 실무자 수기 (SAP/HBR/HSZ export → `01.Source`)
2. **자동 연결** — 클로드 (`X월 v1` 생성)
3. **수기 편집** — 실무자 (환율/배부율 update, 추가/삭제/배부, 사옥운영 대사)
4. **통합** — 실무자 (`26BP+X월 실적_HEV합산ver.` 생성)
5. **Humax 배부판** — 클로드+실무자 (`Humax FC 실적 (X월)`)
6. **EVCS 계정별** — 클로드+실무자 (`EVCS FC 계정별 실적 (X월)`)
7. **Humax 계정별** — 클로드+실무자 (`Humax FC 계정별 실적 (X월)`)

## 5개 에러 (file #1 한계란)

1. 원본 파일 훼손, 복구불가 (신규 생성도 원본 훼손되는 경우)
2. 디자인 변경 / 지시 누락 (재명령)
3. 디자인·조건 10회+ 재요청
4. 토큰 빠르게 소진
5. 작업 후 숫자 검토 부담

## 한계 2종

1. 적요(memo) 봐야 하는 작업 = 클로드 불가 (건별 배부, 항목별 분석)
2. 실질 워킹데이 1일 단축에 머무름

## 진짜 원인 진단

5개 에러 + 한계 모두 **raw 토큰 폭발 + 분할 전략 부재**가 단일 원인. LLM 한계 아님.

- 26BP `예산+실적` = 15,007행 × 63열 = 945K 셀 ≈ 3-5M 토큰
- Claude 컨텍스트 200K (Sonnet 4) 또는 1M (Opus 4.7) → 15-25배 초과
- 후순위 컬럼(C25 적요)이 잘림 → 실무자: "AI가 적요 못 읽음" 오진

**Why:** 1차 강의 후 실무자가 file #1로 적용 시도했으나 1일 단축에 머무름. 한계 원문 인용은 진단 출발점이며, raw 시트 직접 검증으로 오진 입증함.

**How to apply:** Humax 결산 관련 작업 시 file #1을 단일 케이스 스터디 자산으로 활용. 5개 에러 원문은 강의 모듈 1 hook으로 사용. 신규 분석 시 raw 토큰 크기 먼저 측정 후 분할 전략 적용.
