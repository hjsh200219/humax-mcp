# PRODUCT_SENSE.md

> 제품 원칙 + 사용자 + 의사결정 기준.

## 사용자

- **1차**: Humax 경영지원 / 재무 실무자 (Excel 결산 담당, CLI 미경험)
- **2차**: Humax 내부 데이터 엔지니어 (MCP 서버 운영)
- **3차**: 강의 수강생 (Cowork + MCP 모듈)

## 핵심 가치 명제

> "실무자는 CLI를 학습하지 않는다. Claude Desktop에서 자연어로 호출하면 Python 결정론이 처리한다."

학습 비용 0 + 결정론 정확성 동시 달성. 이게 핵심 차별점.

## 안전 정책 P1-P5 (RANKED)

1. **원본 보존 우선** — 모든 다른 가치보다 우선. read-only 기본, write는 명시적 + 백업 강제.
2. **결정론은 Python에 위임** — LLM 산술 금지. 합계/검증/배부 전부 Python.
3. **토큰 효율** — 필터된 응답, 페이지네이션. 사용자 비용 차단.
4. **안전** — 백업 + dry-run + 감사 로그. 실수 복구 가능성.
5. **자연어 호출 가능** — 실무자 CLI 학습 부담 0. 의사결정 시 P5 > 코드 우아함.

충돌 시 위 순서로 우선. 예: P1 vs P5 — 원본 보존이 우선 (read-only 기본).

## 의사결정 기준

새 도구/기능 추가 검토 시:

1. **P1-P5 충돌**: 어떤 P를 약화시키는가?
2. **5개 에러 매핑**: file #1 에러 중 어떤 것을 푸는가? 안 풀면 v0.2+로.
3. **결정론 가능?**: LLM 산술 필요 없는가? 필요하면 Python으로 빼낼 수 있는가?
4. **자연어 호출 직관?**: 도구명/파라미터가 실무자 어휘와 일치하는가?
